    For Image OCR Refinement
When Tesseract.js finishes, the text is often messy (e.g., "Wh4t is 2+2?"). Send this to the AI:

Prompt: "The following text is raw OCR output from a camera image. Clean up the typos, identify the core question, and ignore any background noise text. Return only the sanitized question. Text: [INSERT_OCR_DATA]"

For the "Solver" Logic (Phase 10)
To get the highest accuracy for the Receiver, use a Chain-of-Thought prompt:

Prompt: "You are a professional tutor. Analyze the following question: [INSERT_CLEANED_TEXT].

Briefly explain the concept.

Solve it step-by-step.

Provide the final answer in bold.
Keep the total response under 150 words for mobile readability."

🛠️ Performance Optimization Tips
To ensure Phase 8 and Phase 9 don't crash your server:

Image Pre-processing: In Phase 9, when using sharp, use this specific sequence for the highest OCR accuracy:

JavaScript
sharp(inputBuffer)
  .grayscale()
  .linear(1.5, 0) // Increase contrast
  .resize(2000)   // Upscale if the image is small
  .toBuffer();
Audio Format: Ensure react-mic is saving as MP3 or WAV. If you send a .webm (default in many browsers) to an AI that expects .mp3, it will fail. Use ffmpeg on the backend if necessary.

Socket Heartbeats: Since the audio recording takes 50 seconds, the socket connection might idle out. Ensure your Socket.io config has pingTimeout: 60000 to keep the connection alive during the long recording.

A Note on "Highest Accuracy"
For the Transmitter (Phase 8), use facingMode: "environment" in react-webcam. This forces the phone to use the back camera (higher resolution/autofocus) instead of the selfie camera, which is critical for readable OCR.









Using the Browser's "Web Speech API" (Free)
Most modern browsers (Chrome/Edge) have a built-in feature that converts speech to text for free. You could "turn on" this feature in the Transmitter's browser code.

How it works: As the Transmitter "hears" the 50 seconds of audio, the browser generates a live transcript. When the 50 seconds are up, you send the text string to the backend instead of a heavy audio file.

The Code (npm library): react-speech-recognition.

The Pros: It is 100% free and uses no API keys.

The Cons: It only works while the browser tab is active/awake. If the Transmitter's phone screen turns off, the "captions" stop immediately.
